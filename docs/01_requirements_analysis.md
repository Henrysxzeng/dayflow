# DayFlow · 需求分析报告
**版本：** v1.1  
**日期：** 2026-05-17  
**阶段：** 软件开发生命周期 · 第一阶段

---

## 一、核心问题诊断

现有任务管理工具（滴答清单、Todoist等）的死亡率极高。研究表明 **80% 的用户在下载后 30 天内放弃**。原因不是功能不够，而是：

- **输入成本高，回报感知低**（填了一堆信息，最后还是自己想怎么做就怎么做）
- **计划是静态的**（现实永远打脸计划，打脸后用户放弃的是 App 不是习惯）
- **AI 不知道用户今天有多少时间**（排出来的计划根本做不完，直接放弃）
- **完成感太弱**（没人给你鼓掌）
- **没有与用户建立情感连接**

DayFlow 的核心定位：**让用户感觉每天有个靠谱的人帮他安排，他只需要执行。**

---

## 二、目标用户画像

**主要用户：** 25-40 岁，有一定事业追求但经常感到任务失控的职场人/创业者/自由职业者/学生

**核心痛点：** 任务太多不知道先做什么 → 焦虑拖延 → 什么都没做 → 更焦虑

**他们不缺执行力，缺的是一个每天告诉他"今天就做这几件事"的声音。**

---

## 三、平台选型（关键决策）

### 微信生态的技术壁垒

微信生态对主动推送有极严苛的限制，必须在选型阶段排雷：

| 平台 | 推送能力 | 结论 |
|------|---------|------|
| 订阅号 | 每天仅能群发一次，消息折叠，无法个性化推送 | 不可用 |
| 服务号 | 每月仅能主动推送 4 次（除非用户每天触发互动） | 不可用 |
| **微信小程序 + 订阅消息** | 用户每次完成任务时弹窗申请次日推送授权，可实现每日个性化推送 | **推荐** |
| 原生 App | 完全自由的推送权限，但开发成本高 | 第二阶段考虑 |

### MVP 平台决策：微信小程序

**选择理由：**
- 零摩擦入口（无需下载 App，微信直接打开）
- 订阅消息机制：用户勾选完成任务时，顺势请求"明早推送授权"——完成动作本身就是授权动作，摩擦力极低
- 中国市场覆盖率最高
- 后期可平滑迁移至原生 App

**推送授权设计：**
用户勾选今日最后一个任务时 → 弹窗："今天做到了！授权明早 8 点的计划推送？" → 一键同意 → 获得次日推送权

## 四、功能设计

### MVP 核心功能（第一版必须有）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 快速录入任务 | 对话式输入，名称+截止日期+难度（其他可选），极低摩擦 | P0 |
| 晨间动态问询 | 推送后 AI 先问"今天能用几小时"，再生成计划 | P0 |
| AI 每日计划生成 | 根据可用时间，选出当天 3-5 件最重要的事 | P0 |
| 碎片任务分流 | 预计 ≤10 分钟的任务进入"随手清空"列表，不走 AI 排期 | P0 |
| 一键重排计划 | 临时有变，一键重新生成当日计划 | P0 |
| 任务完成勾选 | 当天任务逐一勾选 | P0 |
| 轻量失败标签 | 未完成任务选原因：太难了 / 没时间 / 不想做 | P0 |
| 智能结转 | 根据失败标签决定：直接结转 / 拆解任务后结转 / 降低优先级 | P0 |
| 订阅消息推送 | 每次勾选任务时申请次日推送授权，保持每日触达 | P0 |

### 第二阶段功能（留存核心）

| 功能 | 说明 |
|------|------|
| 连续天数记录（Streak） | 视觉化"连续高效第 N 天"，含免死金牌机制 |
| 每周 AI 总结报告 | AI 生成"你这周做到了什么"，高光用户成就 |
| 任务完成率趋势图 | 让用户看到自己在进步 |
| 拖延模式分析 | "你经常在周三拖延设计类任务" |
| 能量曲线适配 | 根据用户历史习惯，难事排上午/轻松事排下午 |

### 第三阶段功能（变现核心）

| 功能 | 说明 |
|------|------|
| 目标体系 | 任务 → 项目 → 人生目标，建立长期意义感 |
| 自定义 AI 角色 | 严厉教练 / 温柔助手 / 毒舌朋友 |
| 团队协作 | B 端入口：任务分配、管理者视图 |
| 年度数据报告 | 类 Spotify Wrapped，极强社交传播性 |

## 五、核心业务逻辑

### 任务分流规则

```
新任务录入
    │
    ├─ 预计用时 ≤ 10 分钟 ──→ "随手清空"列表（不走 AI 排期）
    │
    └─ 预计用时 > 10 分钟 ──→ 进入 AI 统筹任务池
                                    │
                                    ↓
                           四象限分类（紧急×重要）
                                    │
                                    ↓
                           结合当日可用时间生成计划
```

### 晨间推送流程

```
早 8 点推送 → "今天有 X 件要事，今天能挤出几小时？"
    │
    ├─ 用户回复时间 ──→ AI 按可用时间重新计算 → 推送最终计划
    │
    └─ 用户 30 分钟不回复 ──→ 按默认 4 小时生成计划自动推送
```

### 未完成任务处理

```
任务未完成
    │
    └─ 用户选择失败原因
            │
            ├─ "太难了" × 连续 2 次 ──→ AI 自动拆解任务粒度
            ├─ "没时间"             ──→ 直接结转，提高次日权重
            └─ "不想做"             ──→ 降低优先级，本周末重新评估
```

## 六、让用户上瘾的机制设计（行为科学）

### 1. 斯金纳箱 × 可变奖励
不是每次完成都有奖励，而是**偶尔出现一个特别好的计划**让用户觉得"这次 AI 真的懂我"。AI 的计划质量要有惊喜感，不能永远中规中矩。

### 2. 塞利格曼的"掌控感"
焦虑的本质是失控感。每天早上的"今日清单"本质上是**每天早上给用户一次掌控感复位**。焦虑者每天都需要它，这个感觉会让人产生依赖。

### 3. Streak 机制（不要打破锁链）
连续天数是已被验证最有效的留存工具（Duolingo 靠此将留存率提升了 3 倍）。
- Streak 的门槛要低：不是"完成所有任务"，而是"每天打开/勾选一个"
- 断掉 Streak 给补救机会（免死金牌），触发损失厌恶但不惩罚用户

### 4. 蔡格尼克效应（未完成任务的心理张力）
人对未完成任务的记忆力是已完成的 2 倍。  
晚间推送不说"今天完成了什么"，而说**"还有 X 件事在等你明天"**——让用户带着轻微心理张力入睡，明早更渴望打开 App。

### 5. 新鲜开始效应（Fresh Start Effect）
每个新起点（周一、月初）人们更愿意改变行为。AI 应识别这些节点，在周一早推送时稍微调整语气，强化"全新一周，重新开始"的仪式感。

### 6. 实施意图（Implementation Intention）
把"我要做 X"变成"我在 X 时间在 X 地点做 X"，完成率提升 300%。  
AI 生成计划时不说"今天做方案"，而说**"上午 10 点，在正式开会前，先用 30 分钟起草方案框架"**。

### 7. 峰终定律（Peak-End Rule）
- **完成当天最后一个任务时**：一句有温度的话，不是普通弹窗
- **每周总结时**：AI 用有感情的语言回顾这周，高光用户的具体成就

### 8. 身份认同绑定
推送语言不说"你今天要做 3 件任务"，而说**"高效的你，今天有 3 件事在等你"**。让用户把"每天完成计划"内化为自我认同的一部分。

### 9. 动态难度校准（流心理学）
契克森米哈赖的心流理论：任务既不太难也不太简单时人会进入心流。  
AI 根据失败标签持续调整计划复杂度，让用户始终处于"有点挑战但能做到"的区间。

## 七、反流失设计

**流失最高危节点：**

| 时间节点 | 原因 | 干预措施 |
|---------|------|---------|
| 第 3 天 | 新鲜感消退 | 主动推送"你的首份效率报告" |
| 第 14 天 | 感觉没进步 | 展示前两周完成任务总数的成就感总结 |
| Streak 断掉后 | 觉得没意义继续 | 免死金牌 + "断掉没关系，重要的是继续" |
| 计划连续失败后 | AI 排的计划做不完 | 自动降低计划强度，先重建信心 |

**极简原则：** 用户每次操作不超过 3 步。录入任务最多 2 步，看计划只需打开 App。

## 八、变现路径

```
免费用户 → 体验价值（养成 Streak 习惯）→ 付费转化
```

| 阶段 | 策略 |
|------|------|
| 冷启动期 | 完全免费，积累口碑和用户行为数据 |
| 增长期 | 订阅制：¥18/月 或 ¥128/年（高级 AI 规划、Streak 恢复、拖延分析等） |
| 成熟期 | 企业版：团队任务分配、管理者视图 |
| 生态期 | 效率课程/教练服务的流量入口 |

**关键变现洞察：** 最愿意付钱的用户是 Streak 超过 30 天的人。前期不急收费，先把这批种子用户找出来，他们转化率极高。

## 九、MVP 范围（第一版严格边界）

**做：**
1. 微信小程序（不做原生 App，降低开发成本）
2. 对话式录入任务（名称 + 截止日期 + 难度，其他可选）
3. 晨间问询可用时间 → AI 生成当日 3 件最重要的事
4. 碎片任务（≤10 分钟）进入"随手清空"列表
5. 勾选完成 + 轻量失败原因标签
6. 一键重新生成当日计划
7. 订阅消息推送（每次勾选任务时授权次日推送）

**不做（第一版砍掉）：**
- Streak 系统
- 数据分析图表
- 多设备同步
- 社交功能
- 任何付费功能

**MVP 验证的核心假设：**  
> 用户愿不愿意每天早上授权推送、看这个计划，并按它行动。  
> 验证了再加功能，假设不成立就调整产品方向。

**DogFooding 原则：**  
开发者本人必须是第一个重度用户。如果产品能让自己每天早上感到"如释重负"，MVP 即告成功，可以推向市场。
