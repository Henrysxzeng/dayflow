## V1.3.0（2026-05-19）— 推送系统重构

### 新功能

**逐任务截止倒计时提醒**
- 每个任务可独立设置截止前提醒（编辑任务 → 开启提醒 → 选择提前时间）
- 定时器每30分钟扫描，精确到分钟的提醒窗口
- 三层渐变文案：≤15分钟冲刺提醒 / ≤60分钟行动催促 / >60小时倒计预警
- 订阅逻辑：新建/编辑任务时同步弹授权，一次授权一条提醒

**每日首次打开自动订阅**
- 每天首次进入 App 自动弹订阅授权（morning + evening + friday 三模板）
- 当天不重复弹窗（localStorage 记录）
- 仅对用户真正接受的模板创建推送记录

**推送文案全面优化**
- 去掉神经科学术语（多巴胺、蔡格尼克效应、前额叶皮层）
- 改为短促行动触发词 + 情绪钩子
- 全部文案 ≤20 字（微信模板限制）

### 修复

- 推送系统全面迁移至直接 WeChat API（修复 `cloud.openapi` access_token 失效问题）
- 修正所有模板字段映射（thing/time 字段类型匹配）
- 修复时区问题：截止时间解析增加 `+08:00`
- `eveningPush` 剩余任务计数改为直接查 tasks 集合（不再依赖 daily_logs 累积数据）
- `formatDeadline` 同日截止不显示"明天"的日历比较修复
- `wx.requestSubscribeMessage` 修复：拆分为同步 handler + 异步 _doSave，保持手势上下文
- 「有事要避开」按钮增加背景样式（不再像普通文字）
- 点击「跳过」不再错误触发 AI 重新规划
- `getTodayPlan` 不再用原始 estimated_minutes 覆盖 AI 已校准的建议结束时间
- `appendToTodayPlan` 增加截止时间约束，避免排到截止后
- 新手引导「跳过」直接关闭引导进入主页

### 技术改动

- 新增云函数：`saveTaskPushAuth`（逐任务推送授权）
- 新增云函数：`deadlineReminder`（每30分钟扫描，推送即将截止的任务提醒）
- `push_auth` 集合新增 `task_id` 字段（逐任务推送记录）
- `savePushAuth` 移除 deadline 记录创建，新增 friday 记录
- 全部推送函数用 `https` 模块直调微信 API（替代 `cloud.openapi.subscribeMessage.send`）
- 前端 `add-task.js` 重构为同步 handler + 异步 _doSave 模式
